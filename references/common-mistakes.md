# Common Mistakes / 常见坑

Things that trip up almost every new OpenClaw user.
几乎每个新用户都会踩的坑。

---

## ❌ "I want to set up everything on Day 1" / "第一天就全配好"

**What happens:** 4 hours configuring skills, cron, multiple channels... then overwhelmed when something breaks.
花 4 小时配 skill、cron、多频道……然后一个东西坏了就崩溃。

**Do instead:** Minimal setup → use a few days → add one thing → repeat.
先最小配置 → 用几天 → 加一个东西 → 重复。

---

## ❌ "I wrote SOUL.md once and never touched it" / "SOUL.md 写完就不管了"

**What happens:** Agent personality feels "off" but you can't tell why.
agent 性格总感觉不对，但说不上来哪里不对。

**Do instead:** Revise every 2-3 days. The 10th version will feel completely different from the 1st.
每 2-3 天改一次。第十版跟第一版完全不一样。

---

## ❌ "I skipped the memory system" / "没搞记忆系统"

**What happens:** Every conversation starts from zero. It feels like a chatbot.
每次对话从零开始，跟普通聊天机器人没区别。

**Do instead:** Set up daily notes from Day 2. Even minimal notes compound over time.
第二天就开始写日记。哪怕很简短，时间长了效果惊人。

---

## ❌ "I copied someone else's entire config" / "照抄别人的配置"

**What happens:** Rules conflict with YOUR workflow. You fight the config instead of using it.
规则跟你的习惯冲突，你在跟配置打架而不是用它。

**Do instead:** Start minimal. Read others for inspiration, adapt to YOUR workflow.
从最简开始。看别人的配置找灵感，但要适配自己的工作流。

---

## ❌ "I treat the agent like a search engine" / "把 agent 当搜索引擎"

**What happens:** You only ask factual questions. The agent never develops personality.
你只问事实性问题，agent 永远没有性格。

**Do instead:** Share context. Say WHY you're asking. "我在准备跟老板汇报这个项目，帮我想想风险" vs "列出项目风险"。

---

## ❌ "I automated too much too fast" / "自动化搞太多太快"

**What happens:** Cron firing everywhere, unexpected notifications, chaos. You turn everything off.
cron 到处触发，通知乱飞。你把所有东西都关了。

**Do instead:** One automation at a time. Run it for a week. Keep it if useful, remove if not.
一次加一个自动化。跑一周。有用就留，没用就删。

---

## ❌ "I never gave feedback" / "从来不给反馈"

**What happens:** Agent does mediocre work forever because it doesn't know your standards.
agent 永远做得一般般，因为它不知道你的标准。

**Do instead:** Be specific. "这个总结结构很好" or "太啰嗦了，我要三句话的版本"。

---

## ❌ "I expected it to be perfect immediately" / "期待它立刻完美"

**What happens:** First interaction isn't amazing. You conclude "it's not useful."
第一次对话不惊艳，你觉得"没用"。

**Do instead:** Give it a week. Day 1 is setup. Week 1 is calibration. Real value shows up in week 2-3.
给它一周。第一天是安装，第一周是磨合。真正的价值在第 2-3 周出现。
